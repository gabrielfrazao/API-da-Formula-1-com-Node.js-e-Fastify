🚀 COMO USAR
═══════════════════════════════════════════════════════════════════════════════

1️⃣  INSTALAR DEPENDÊNCIAS
   $ npm install

2️⃣  CONFIGURAR VARIÁVEIS DE AMBIENTE
   $ cp .env.example .env

3️⃣  INICIAR SERVIDOR (DESENVOLVIMENTO)
   $ npm run dev
   
   OU PRODUÇÃO:
   $ npm start

4️⃣  ACESSAR A API
   Browser: http://localhost:3000
   Health:  http://localhost:3000/health
   Docs:    Ver README.md


═══════════════════════════════════════════════════════════════════════════════
📡 EXEMPLOS DE REQUISIÇÕES
═══════════════════════════════════════════════════════════════════════════════

🔵 LISTAR TODOS OS PILOTOS
   curl http://localhost:3000/drivers

🔵 FILTRAR PILOTOS FERRARI
   curl "http://localhost:3000/drivers?team=Ferrari&sort=points"

🔵 OBTER PILOTO POR ID
   curl http://localhost:3000/drivers/1

🔵 CRIAR NOVO PILOTO
   curl -X POST http://localhost:3000/drivers \
     -H "Content-Type: application/json" \
     -d '{
       "name": "Sebastian Vettel",
       "team": "Aston Martin",
       "nationality": "German",
       "number": 33,
       "points": 180,
       "championships": 4
     }'

🔵 RANKING DE PILOTOS
   curl http://localhost:3000/drivers/ranking/points

🔵 LISTAR TIMES
   curl http://localhost:3000/drivers/teams

🔵 ATUALIZAR PILOTO
   curl -X PUT http://localhost:3000/drivers/1 \
     -H "Content-Type: application/json" \
     -d '{"points": 500}'

🔵 REMOVER PILOTO
   curl -X DELETE http://localhost:3000/drivers/9

🔵 HEALTH CHECK
   curl http://localhost:3000/health

🔒 SEGURANÇA IMPLEMENTADA
═══════════════════════════════════════════════════════════════════════════════

HELMET.JS
├─ XSS Protection ...................... X-XSS-Protection header
├─ Content Security Policy ............. CSP header
├─ Clickjacking Protection ............. X-Frame-Options: deny
├─ MIME Type Sniffing Protection ....... X-Content-Type-Options: nosniff
├─ HSTS ................................ HTTP Strict Transport Security
└─ Referrer Policy ..................... Leak prevention

CORS
├─ Whitelist de origens ................. Configurável via .env
├─ Métodos permitidos ................... GET, POST, PUT, DELETE, OPTIONS
├─ Headers customizados ................. Content-Type, Authorization
└─ Credentials .......................... true/false

RATE LIMITING
├─ Limite de requisições ................ 100 por 15 minutos
├─ Whitelist de IPs ..................... 127.0.0.1
├─ Skip para health checks .............. /health não contabiliza
└─ Proteção DDoS ....................... Ativada

INPUT VALIDATION
├─ Sanitização de strings ............... Remove tags HTML
├─ Validação de tipos ................... Previne type coercion
└─ Prevenção de injeção ................. SQL injection protection

ERROR HANDLING
├─ Não expõe detalhes internos .......... Respostas controladas
├─ Request IDs para tracking ............ Identificação de requests
└─ Logging detalhado .................... Server-side only

DEPLOYMENT
├─ Docker non-root user ................. Segurança de container
├─ Multi-stage build .................... Otimização de imagem
└─ HTTPS Ready .......................... Suporta certificados SSL
